## Раздел 1 — Проверка на твоём iPhone 17 Pro Max (device-only)

Эти три гейта **физически невозможно** проверить на симуляторе — нужен только
твой айфон.

### F6 — Crash-free 60 минут непрерывной транскрипции
**Зачем:** доказать стабильность памяти/audio session за длинный сеанс.

**Что делать:**
1. На маке: `cd ~/projects/dspeech-ios && open Dspeech.xcodeproj`
2. В Xcode выбери схему **Dspeech**, destination **твой iPhone 17 Pro Max**
   (он должен быть подключён по USB-C или Wi-Fi, и доверен).
3. Cmd+Shift+K (чистая сборка), потом Cmd+R (запуск на устройстве).
4. На айфоне открой приложение, дай permissions (Speech recognition,
   Microphone) — в первый запуск.
5. Нажми кнопку записи. Говори / включи фоновый источник речи (подкаст с
   AirPods / громкая колонка рядом). Не выгружай приложение из памяти.
6. Через 60 минут — посмотри: приложение не упало, транскрипт продолжает
   обновляться.

**Принимаем:** 60 минут без падений, без OOM, транскрипт жив.

**Куда записать результат:** в Notion в карточке этой задачи: дата теста, длительность реально прошедшая, упало/не упало, скриншот последнего экрана.

### F7 — Расход батареи ≤25% в час
**Зачем:** доказать что приложение не уничтожает батарею (приёмка по PRD §F7).

**Что делать:**
1. Зарядить айфон до 100%. Включить **Авиарежим** (чтобы не было сетевых
   нагрузок — речь обрабатывается локально).
2. Запустить Dspeech в режиме записи (как в F6).
3. Подержать 60 минут.
4. Посмотреть в **Настройки → Аккумулятор** — расход батареи за последний
   час по приложению Dspeech.

**Принимаем:** Dspeech ≤25% за 60 минут.

**Куда записать:** скриншот **Настройки → Аккумулятор → Последние 24 часа**
с раскрытой подробной разбивкой по часам.

---

### F8 — Корректная остановка в фоне
**Зачем:** при сворачивании приложения запись должна корректно прерываться,
а при возврате — состояние должно быть как пользователь ожидает.

**Что делать:**
1. Запусти Dspeech, начни запись.
2. Сверни приложение (свайп вверх).
3. Подожди 60 секунд.
4. Раскрой приложение обратно.
5. Проверь: запись либо корректно остановлена с сохранённой последней
   фразой, либо — если она продолжается — fed-back индикатор корректно
   обновляется.

**Принимаем:** никаких подвисших audio session, никаких потерь транскрипта.

**Куда записать:** видео-запись цикла через **Запись экрана** на самом
айфоне (Control Center → круглая кнопка записи экрана).

## Раздел 4 — Статус автопилота и pipeline

### Что сейчас работает
- **Lane A**: workday-pilot-v2.sh — 5-фазный sequential+parallel конвейер.
  - P1 критический путь (W4b-r4 → W7 → W8 → W9), sequential
  - P2 хардненинг (W10/W11/W12/W13), parallel x4 — четыре `claude -p` одновременно
  - P4 polish (W14 → W15 → W16), sequential — Liquid Glass → a11y → Gemini iteration
  - P5 финал (W17), sequential — merge + docs + push
- **watchdog-v3.sh** — пассивный наблюдатель (не respawn-цикл), реагирует
  только на смерть пилота.
- **Rate-limit-aware sleep** — пилот парсит "resets HH:MMam (TZ)" и спит
  ровно до сброса +90s. Никаких бешеных respawn.
- **Codex fallback** — заложен в код, активируется когда Codex CLI появится
  на mac24 (сейчас нет — нужен npm/brew).
- **Expert review framework** — `.agent-prompts/expert-review-framework.md`,
  обязательный к применению каждым ревьюером (Sendable, ошибки, a11y,
  perf, security, тесты, anti-rubber-stamp).

### Что развивается дальше
1. **Установка Codex CLI на mac24** — fallback на gpt-5.4 когда Claude rate-limit.
   Сейчас не установлен (brew/npm отсутствуют). Если поставишь homebrew —
   автопилот сам инсталлит Codex.
2. **Параллелизация polish lane** — сейчас sequential для безопасности
   (один файл могут ревизить); если разделим скоп — можно делать || x3.
3. **Property-based testing для большего покрытия** — Hypothesis-style
   тесты для Core/Translation и Core/Audio.
4. **Snapshot testing** — добавим snapshot-tests для UI (см.
   `docs/snapshot-tests-design.md`).
5. **CI на GitHub Actions** — пока на mac24 локально. Можно потом
   зеркалить в GHA с self-hosted runner на mac24.

### Что от тебя ожидается только опционально
- Поставить homebrew на mac24 (для Codex CLI fallback): `/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"`. После этого автопилот сам поставит `npm` и `codex`.
