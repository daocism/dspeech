# App Store screenshot plan

Status: staged capture plan. This document defines simulator states only; it
does not upload screenshots or update App Store Connect.

## Source requirements

Apple's screenshot specification page currently lists iPhone 6.9-inch and
6.5-inch classes plus iPad 13-inch and 12.9-inch classes. The orchestrator brief
uses "6.7-inch"; this plan preserves that label as `iphone-67` while targeting
Apple's current 6.9-inch accepted pixel sizes.

Source: https://developer.apple.com/help/app-store-connect/reference/app-information/screenshot-specifications

## Capture matrix

| Plan key | App Store class | Accepted portrait output | Simulator candidates |
|---|---|---:|---|
| `iphone-67` | iPhone 6.9-inch class, replacing the stale 6.7-inch label | `1320x2868`, `1290x2796`, or `1260x2736` | `iPhone 17 Pro Max`, `iPhone 16 Pro Max`, `iPhone 15 Pro Max`, `iPhone 14 Pro Max`, `iPhone 13 Pro Max` |
| `iphone-65` | iPhone 6.5-inch class | `1284x2778` or `1242x2688` | `iPhone 11 Pro Max`, `iPhone XS Max` |
| `ipad-13` | iPad 13-inch class | `2064x2752` or `2048x2732` | `iPad Pro 13-inch (M4)`, `iPad Pro (13-inch) (M4)`, `iPad Pro 13-inch (M5)` |
| `ipad-129` | iPad Pro 12.9-inch class | `2048x2732` | `iPad Pro (12.9-inch) (6th generation)`, `iPad Pro (12.9-inch) (5th generation)`, `iPad Pro (12.9-inch) (4th generation)`, `iPad Pro (12.9-inch) (3rd generation)`, `iPad Pro (12.9-inch) (2nd generation)` |

## Mandatory state

Each device class captures one mandatory state:

`01-cockpit-local-empty`

- Launch arguments:
  - `-dspeech.privacy.mode.v1 localOnly`
  - `-dspeech.voicefilter.modelpack.v1 absent`
- Required visible content:
  - app name;
  - visible `LOCAL` processing badge;
  - route-health indicator;
  - translation toggle;
  - primary start control;
  - empty-state copy that implies on-device processing without claiming
    certified avionics behavior.
- Prohibited visible content:
  - cloud transcription claims;
  - analytics, tracking, or advertising claims;
  - hardware compatibility promises;
  - pricing or StoreKit language;
  - outbound contact prompts.

## Output contract

The script writes idempotently to:

`tmp/app-store-screenshots/<date>/<plan-key>/01-cockpit-local-empty.png`

The date folder is generated by the capture script. Re-running the script for
the same date overwrites the same files.

The script is no-outbound by default. It fails before `xcodebuild` if the
configured DerivedData folder does not already contain local SwiftPM checkouts
for the packages pinned in `Package.resolved`.

## Release gates outside this script

- Final app icon and styled screenshot frame assets are not present in this
  repository slice.
- App Store Connect upload is blocked on Andrei-held Apple Developer access.
- Privacy labels must be approved separately before publication.
