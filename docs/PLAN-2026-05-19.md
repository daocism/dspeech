## Anti-AI-coding-failure-mode guards (each wave gets these in its dispatch prompt)

1. **No hallucinated APIs.** Every Apple framework method call must be verified before write ‚Äî `Context7 /websites/developer_apple` query, or `grep -rn` of an existing usage in the repo, OR fail with "API not verifiable, halting".
2. **No placeholders.** `TODO`, `fatalError("unimplemented")`, "Coming soon", commented-out stubs ‚Üí reviewer auto-rejects. Verifier greps and fails build report.
3. **No silent failures.** Throw with context. No `try?` swallowing; no `catch { return nil }`. Errors propagate to a single boundary per surface (ViewModel or scene).
4. **No scope drift.** Each implementer owns its declared paths only. Editing outside that set is a reviewer block.
5. **No half-implementation.** Wave is "done" only when its owned files compile, its owned tests are green via `xcodebuild test`, and `git diff` reads coherently. Otherwise revert.
6. **No echo-chamber reviewer.** Reviewer runs on ubuntu-vm in a fresh `claude -p` session that has never seen implementer transcripts. Reviewer prompt is skeptical-by-default; must produce ‚â•3 concrete concerns or an APPROVE-with-rationale.
7. **Verification ran, not "assumed".** W7 gate runs the FULL `xcodebuild build test`, captures the result file, greps for forbidden tokens, and ships screenshots. No declaration of done without those artifacts on disk.
8. **Parallel agents fighting same file.** Strict file partition above. Mid-wave conflict ‚Üí tech-lead reassigns, no auto-merge.
9. **Context loss across handoff.** Every wave appends a structured block to `handoff.md` with `files_changed`, `tests_run`, `errors`, `next_steps`.
10. **Anti-cycle (review ‚Üî fix).** Reviewer-implementer disagreement: max 3 rounds; round 4 escalates to tech-lead, round 5 to Andrei. Identical re-fix twice = replace the implementer agent.
